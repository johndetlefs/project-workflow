Findings:
- Owner approval is described as a single requirements/AC authority envelope, not repeated approval for unchanged work.
- Epic guidance routes through EPIC-CONTRACT.md, DECOMPOSITION.md, AMENDMENTS.md, adoption, child charters, proof recipes, and closeout gates.
- Task guidance routes new tasks through requirements confirmation and task approve-requirements before implementation-oriented work.
- Requirements and planner guidance require proof-recipe selection when work claims visual/reference fidelity, runtime/deployment/source proof, external contract alignment, or multi-context behavior.
- Implement and QA guidance require recipe-triggered structured evidence in EVIDENCE.json and reject prose, code review, tests, builds, surrogate artifacts, and wrong target/source pairs as substitutes where recipes require stronger proof.
- Cursor rules mirror the same gates for non-Codex entry points.
